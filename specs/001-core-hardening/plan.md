# Implementation Plan: Core Hardening — Multi-Agent Pipeline, API & Ollama Integration

**Branch**: `feature/issue-1-2-3-core-hardening` | **Date**: 2026-07-08 | **Spec**: [spec.md](spec.md)

**Closes**: #1, #2, #3

## Summary

Harden the scaffolded Six Thinking Hats AI system across three tightly coupled layers: the multi-agent pipeline (six hat agents + controller), the FastAPI REST interface, and the Ollama LLM integration. Key additions: robust per-hat error handling with graceful degradation, whitespace-aware input validation, startup connectivity probe, structured logging, and configurable Ollama backend via environment variables.

## Technical Context

**Language/Version**: Python 3.11

**Primary Dependencies**: FastAPI 0.111, Uvicorn 0.30, httpx 0.27, Pydantic v2, python-dotenv

**Storage**: N/A — stateless per-request pipeline

**Target Platform**: Linux server (local dev + Docker)

**Project Type**: Web service (REST API)

**Performance Goals**: Response within Ollama generation time; no added latency from the pipeline layer itself

**Constraints**: 120s Ollama read timeout; 5s connect timeout; must start even if Ollama is unreachable

**Scale/Scope**: Single-instance; no horizontal scaling concerns in this phase

## Implementation Design

### `models.py` changes

Add `error: bool = False` field to `HatAnalysis`. Add `@field_validator("problem")` to `ProblemInput` that strips and rejects empty strings.

### `agents/base.py` changes

- Split `httpx.Timeout`: `connect=5.0`, `read=120.0`
- Wrap `_call_ollama` body in try/except; re-raise for controller to catch
- Add structured log calls: `hat_call_start`, `hat_call_done` (with `duration_ms`, `error`)
- `analyze()` catches all exceptions → returns `HatAnalysis(error=True, response=str(exc))`

### `controller.py` changes

- Switch to `asyncio.gather(*tasks, return_exceptions=True)`
- Post-gather: convert any `Exception` result to `HatAnalysis(error=True, response=str(exc))`
- Build `aggregated` context only from non-error analyses (or include error note)
- Blue Hat still always runs

### `main.py` changes

- Load `.env` via `python-dotenv` at module import
- Replace bare `app = FastAPI(...)` with a `lifespan` context manager that probes `GET /api/tags` on Ollama at startup
- Add request logging middleware (log `method`, `path`, `duration_ms`, `status_code`)
- Keep `POST /analyze` and `GET /health` routes unchanged in signature

### Logging setup

Add `logging_config.py` with `dictConfig` defining a JSON-ish formatter for stdout. Import and apply in `main.py` before app creation.
